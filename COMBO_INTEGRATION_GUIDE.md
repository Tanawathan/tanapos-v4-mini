## 🚀 立即測試步驟

### 第一步：執行升級SQL
1. 登入 Supabase Dashboard
2. 進入 SQL Editor
3. 執行 `upgrade-combo-system.sql` 檔案內容
4. 確認以下表格和資料創建成功：
   - `combo_choices` 表格
   - 夏日特色套餐和經典商務套餐
   - 套餐選擇規則

### 第二步：測試套餐功能
1. **開啟 SimplePOS**
   - 瀏覽器：http://localhost:5173
   - 點擊進入點餐介面

2. **切換到套餐模式**
   - 點擊「🍽️ 套餐組合」按鈕
   - 查看可用套餐列表

3. **測試固定套餐**
   - 點擊「經典雞排套餐」
   - 應該直接加入購物車

4. **測試可選套餐**
   - 點擊「夏日特色套餐」或「經典商務套餐」
   - 應該開啟套餐選擇器
   - 選擇前菜、主餐、飲品、甜點
   - 確認後加入購物車

## 🎯 套餐功能特色

### 🔄 模式切換
- **🍔 單點商品**：傳統單品點餐
- **🍽️ 套餐組合**：套餐式點餐，顯示可用套餐數量

### 🎨 套餐類型視覺區分
- **固定套餐**：橙色「固定套餐」標籤，直接加入購物車
- **可選套餐**：橙色「可選套餐」標籤，開啟選擇器

### 💰 動態價格計算
- 基本套餐價格 + 選擇項目價格
- 實時顯示總價格
- 支援多數量套餐

### 📱 套餐選擇器功能
- **分類選擇**：按分類顯示可選項目
- **數量控制**：最少/最多選擇限制
- **視覺提示**：已選項目顯示勾選標記
- **價格顯示**：每項附加價格清楚標示

## 📊 資料庫架構

### combo_products (套餐主表)
- `combo_type`: 'fixed' | 'selectable'
- 固定套餐直接結帳，可選套餐需要選擇

### combo_choices (套餐選擇規則)
- 定義每個可選套餐的分類選擇規則
- 控制最少/最多選擇數量

### 示例套餐配置
```sql
-- 夏日特色套餐 (可選套餐)
可選 1 個前菜 (涼拌系列)
可選 1 個主餐 (海鮮類)  
可選 1 個飲品 (冷飲)
可選 1 個甜點 (冰品)

-- 經典商務套餐 (可選套餐)  
可選 1 個前菜 (熱食開胃菜)
可選 1 個主餐 (肉類主食)
可選 1 個飲品 (熱飲或冷飲)
可選 1 個甜點 (精緻甜點)
```

## 🔧 故障排除

**如果套餐不顯示：**
1. 確認 `upgrade-combo-system.sql` 已執行
2. 檢查 combo_products 表格有資料
3. 確認套餐 `is_available = true`

**如果選擇器不開啟：**
1. 確認套餐 `combo_type = 'selectable'`
2. 檢查 combo_choices 表格有對應規則
3. 查看瀏覽器控制台錯誤訊息

**如果價格計算錯誤：**
1. 確認產品價格設定正確
2. 檢查套餐基本價格
3. 驗證選擇項目的附加價格
