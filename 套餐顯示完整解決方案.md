3. **查看 Network 頁籤**，確認 API 請求成功

### 常見問題：
1. **Supabase RLS 政策**：確認套餐表的 RLS 政策正確
2. **資料庫連接**：確認 .env 檔案中的 Supabase 設定
3. **快取問題**：嘗試硬重新整理（Ctrl+Shift+R）

## 📝 技術細節

### 系統架構：
```
POS 介面 → store-supabase.ts → api.ts → Supabase Database
                ↓                ↓           ↓
           loadProducts()   getAll()     products + combo_products
                ↓                ↓           ↓
            合併顯示      ← 產品轉換 ←    分類整合
```

### 資料流：
1. POS 載入時調用 `loadProducts()`
2. `api.ts` 同時查詢 `products` 和 `combo_products` 表
3. 套餐轉換為 Product 格式並添加 🍽️ 標識
4. 按分類過濾時包含該分類的套餐
5. 在 UI 中統一顯示
