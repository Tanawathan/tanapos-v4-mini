# 🎯 TanaPOS v4 完整工作流程指南

## 📋 **正確的餐廳營運工作流程**

### 1. **桌台管理** 📊
**路由**: `/tables`

**功能**:
- 查看所有桌台狀態
- 即時顯示桌台關聯的訂單
- 手動更新桌台狀態（可使用、使用中、清理中、已預約）

**狀態說明**:
- 🟢 **可使用** - 桌台空閒，可安排客人
- 🔴 **使用中** - 桌台有客人，可能有訂單
- 🟡 **清理中** - 桌台需要清理
- 🟣 **已預約** - 桌台已被預約
- ⚫ **故障** - 桌台無法使用

---

### 2. **點餐流程** 🛒
**路由**: `/pos-simple`

#### **步驟 A: 選擇桌台**
1. 在點餐系統頂部選擇桌台號碼
2. 確保選擇的桌台狀態為"可使用"

#### **步驟 B: 添加商品**
1. 瀏覽產品分類
2. 點擊產品卡片加入購物車
3. 使用 📝 按鈕為商品添加特殊要求

#### **步驟 C: 確認訂單**
1. 檢查購物車內容和總金額
2. 確認桌台號碼正確
3. 點擊"結帳"按鈕

#### **步驟 D: 系統自動處理**
- ✅ 創建訂單記錄
- ✅ 自動將桌台狀態改為"使用中"
- ✅ 生成訂單號碼
- ✅ 清空購物車

---

### 3. **訂單追蹤** 📋
**路由**: `/orders`

**功能**:
- 查看所有訂單
- 更新訂單狀態
- 訂單詳細信息

**訂單狀態流程**:
```
待處理 → 已確認 → 準備中 → 就緒 → 已送達 → 已完成
```

---

### 4. **廚房顯示系統 (KDS)** 👨‍🍳
**路由**: `/kds`

**功能**:
- 顯示需要處理的訂單
- 更新料理狀態
- 按優先級排序

---

### 5. **完整工作流程示例** 🔄

#### **場景：新客人到店**

1. **接待客人** 
   - 打開桌台管理 (`/tables`)
   - 查看可用桌台
   - 引導客人入座

2. **點餐**
   - 打開點餐系統 (`/pos-simple`)
   - 選擇對應桌台號碼
   - 添加客人想要的商品
   - 為特殊需求添加備註
   - 點擊結帳完成訂單

3. **廚房處理**
   - 廚房查看 KDS (`/kds`)
   - 處理訂單項目
   - 更新料理狀態

4. **服務送餐**
   - 查看訂單管理 (`/orders`)
   - 確認料理完成
   - 送餐到桌台

5. **結帳離店**
   - 客人用餐完成
   - 進行最終結帳
   - 桌台管理中釋放桌台
   - 標記為清理中，等待下一組客人

## 🔧 **技術整合說明**

### **數據同步**
- ✅ 所有功能模組使用同一個 Supabase 數據庫 (`store-supabase`)
- ✅ 桌台狀態與訂單自動關聯
- ✅ 即時數據更新
- ✅ 頁面刷新時自動重新載入數據

### **組件數據載入**
- **點餐系統** (`/pos-simple`): 自動載入產品和分類
- **桌台管理** (`/tables`): 自動載入桌台和訂單
- **訂單管理** (`/orders`): 自動載入所有訂單
- **廚房顯示** (`/kds`): 自動載入活躍訂單

### **關鍵字段映射**
- 訂單使用 `table_number` 關聯桌台
- 桌台使用 `table_number` 作為識別
- 系統自動處理 ID 對應關係

### **工作流程驗證**
1. ✅ 點餐系統創建訂單後自動佔用桌台
2. ✅ 桌台管理顯示關聯訂單
3. ✅ 訂單狀態與桌台狀態同步
4. ✅ 所有模組使用真實數據庫
5. ✅ 頁面刷新後數據持續可見

## 🎉 **測試步驟**

1. **打開桌台管理**: `http://localhost:5173/tables`
   - 確認能看到桌台列表
   - 確認顯示"✅ 已連接數據庫"

2. **打開點餐系統**: `http://localhost:5173/pos-simple`
   - 選擇一個桌台（例如：桌台 1）
   - 添加幾個商品到購物車
   - 點擊結帳

3. **回到桌台管理**: 刷新頁面
   - 確認桌台 1 狀態變為"使用中"
   - 確認能看到剛才創建的訂單

4. **打開訂單管理**: `http://localhost:5173/orders`
   - 確認能看到剛才創建的訂單
   - 確認訂單詳細信息正確
