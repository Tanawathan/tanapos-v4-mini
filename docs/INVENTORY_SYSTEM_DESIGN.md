# 🍽️ TanaPOS V4-Mini 庫存與食譜管理系統架構設計

## 📋 系統架構概覽

### 🗂️ 數據層級結構
```
原物料 (Raw Materials)
    ↓ (透過食譜製作)
半成品 (Semi-finished Products) ←─┐
    ↓ (透過食譜組合)              │
成品/菜單項目 (Finished Products) ←─┘
    (可能直接使用原物料)
```

**彈性組合模式:**
- 原物料 → 半成品 → 成品 (預製模式)
- 原物料 + 半成品 → 成品 (混合模式)
- 原物料 → 成品 (直接模式)

### 🏗️ 核心組件架構

#### 1. 原物料管理 (Raw Materials Management)
```typescript
interface RawMaterial {
  id: string
  name: string                    // 食材名稱 (如: 豬肉片、米、蛋)
  category: string               // 分類 (肉類、蔬菜、調料等)
  unit: string                   // 單位 (公斤、公升、包)
  currentStock: number           // 當前庫存
  minStock: number              // 最低庫存警戒線
  maxStock: number              // 最大庫存
  cost: number                  // 成本價格
  supplier: string              // 供應商
  expiryDate?: Date             // 保存期限
  storageLocation: string       // 儲存位置
  lastRestockDate: Date         // 上次進貨日期
}
```

#### 2. 半成品管理 (Semi-finished Products)
```typescript
interface SemiFinishedProduct {
  id: string
  name: string                   // 半成品名稱 (如: 打拋豬、荷包蛋、蔬菜盤)
  category: string              // 分類 (主料、配菜、醬料等)
  unit: string                  // 單位 (份、盤、碗)
  // 實際庫存 (已製作的半成品)
  actualStock: number           // 實際庫存數量
  minActualStock: number       // 最低實際庫存
  // 虛擬庫存 (從原料計算可製作的數量)
  virtualStock: number         // 可製作數量 (自動計算)
  totalAvailable: number       // 總可用數量 (實際+虛擬)
  
  preparationTime: number      // 製作時間 (分鐘)
  shelfLife: number           // 保存時間 (小時)
  actualCost: number          // 實際製作成本 (已製作的)
  virtualCost: number         // 虛擬製作成本 (從原料計算)
  recipe: Recipe              // 製作食譜
  autoRestock: boolean        // 是否自動補製
  restockThreshold: number    // 自動補製閾值
}
```

#### 3. 成品/菜單項目 (Menu Items)
```typescript
interface MenuItem {
  id: string
  name: string                  // 菜品名稱 (如: 打拋豬飯)
  category: string             // 分類 (主餐、小食、飲品)
  price: number               // 售價
  
  // 成本計算 (動態計算)
  totalCost: number           // 總成本 (自動計算)
  rawMaterialCost: number     // 原料成本
  semiProductCost: number     // 半成品成本
  margin: number              // 利潤率
  
  // 庫存與可用性
  actualStock: number         // 實際庫存 (預製成品)
  virtualStock: number        // 虛擬庫存 (可從材料製作的數量)
  totalAvailable: number      // 總可供應數量
  
  preparationTime: number     // 出餐時間
  recipe: Recipe              // 組合食譜
  isAvailable: boolean        // 是否可供應 (自動計算)
  availabilityReason?: string // 不可供應原因
}
```

#### 4. 食譜系統 (Recipe System)
```typescript
interface Recipe {
  id: string
  name: string                 // 食譜名稱
  type: 'raw_to_semi' | 'semi_to_menu' | 'mixed_to_menu'  // 食譜類型
  ingredients: RecipeIngredient[]        // 所需材料
  instructions: string[]                 // 製作步驟
  yield: number                         // 產出數量
  preparationTime: number               // 製作時間
  difficulty: 'easy' | 'medium' | 'hard' // 難度等級
  
  // 成本計算相關
  costCalculation: 'auto' | 'manual'    // 成本計算方式
  manualCost?: number                   // 手動設定成本
  laborCost?: number                    // 人工成本
  overheadCost?: number                 // 間接成本
}

interface RecipeIngredient {
  id: string                   // 材料ID
  type: 'raw' | 'semi'        // 材料類型
  quantity: number            // 所需數量
  unit: string               // 單位
  notes?: string             // 備註 (如: 切片、調味等)
  isOptional: boolean        // 是否為可選材料
  substitutes?: string[]     // 可替代材料ID列表
}

// 庫存計算服務
interface StockCalculationService {
  // 計算半成品虛擬庫存
  calculateSemiVirtualStock(semiId: string): number
  
  // 計算成品虛擬庫存
  calculateMenuVirtualStock(menuId: string): number
  
  // 計算製作成本
  calculateRecipeCost(recipeId: string): {
    rawMaterialCost: number
    semiProductCost: number
    totalCost: number
  }
  
  // 檢查是否可製作
  canMake(itemId: string, quantity: number): {
    canMake: boolean
    missingIngredients: Array<{
      id: string
      name: string
      required: number
      available: number
      shortfall: number
    }>
  }
}
```

## 🔄 業務流程設計

### 流程 1: 原物料到半成品
```
1. 原物料庫存檢查
2. 半成品需求預測 (基於實際庫存不足)
3. 製作計劃生成
4. 原物料扣庫
5. 半成品製作
6. 半成品入庫 (實際庫存增加)
7. 成本核算
8. 虛擬庫存重新計算
```

### 流程 2: 混合模式成品製作 (原料+半成品)
```
1. 訂單接收
2. 材料可用性檢查:
   - 檢查半成品實際庫存
   - 檢查原料庫存
   - 計算虛擬可製作數量
3. 庫存扣除:
   - 優先使用半成品實際庫存
   - 不足時使用原料製作
4. 成品組合製作
5. 出餐
6. 成本結算 (分層計算)
```

### 流程 3: 智能庫存補充流程
```
1. 即時庫存監控 (實際+虛擬)
2. 警告觸發:
   - 實際庫存不足警告
   - 虛擬庫存不足警告 (原料短缺)
3. 補貨建議:
   - 半成品製作建議
   - 原料採購建議
4. 執行補充
5. 庫存更新 (實際+虛擬重算)
```

### 流程 4: 成本計算流程
```
1. 原料成本追蹤
2. 半成品成本計算:
   - 實際製作成本
   - 虛擬製作成本 (從當前原料價格)
3. 成品成本組合:
   - 直接原料成本
   - 半成品成本 (實際或虛擬)
   - 人工成本
   - 間接成本
4. 動態成本更新
```

## 💡 實際業務案例說明

### 案例：打拋豬飯製作

#### 材料層級結構：
```
原物料:
├── 豬肉片 (500g庫存)
├── 九層塔 (100g庫存) 
├── 米 (2kg庫存)
├── 雞蛋 (10顆庫存)
├── 小黃瓜 (5條庫存)
└── 番茄 (8顆庫存)

半成品:
├── 打拋豬 (實際庫存: 2份, 虛擬庫存: 5份 = 總計7份可用)
├── 白飯 (實際庫存: 10份, 虛擬庫存: 20份 = 總計30份可用)
├── 荷包蛋 (實際庫存: 3份, 虛擬庫存: 10份 = 總計13份可用)
└── 蔬菜盤 (實際庫存: 0份, 虛擬庫存: 5份 = 總計5份可用)

成品:
└── 打拋豬飯 (虛擬庫存: 5份 - 受限於蔬菜盤)
```

#### 訂單處理流程：
1. **客戶點餐**: 打拋豬飯 x 3份
2. **庫存檢查**:
   - 打拋豬: 需要3份，有7份可用 ✅
   - 白飯: 需要3份，有30份可用 ✅  
   - 荷包蛋: 需要3份，有13份可用 ✅
   - 蔬菜盤: 需要3份，有5份可用 ✅
3. **製作流程**:
   - 使用實際庫存: 打拋豬2份 + 白飯3份 + 荷包蛋3份
   - 現製: 打拋豬1份 (從原料) + 蔬菜盤3份 (從原料)
4. **成本計算**:
   - 打拋豬: 2份×實際成本 + 1份×虛擬成本
   - 其他: 按實際/虛擬成本分別計算

#### 庫存更新後：
```
半成品剩餘:
├── 打拋豬 (實際: 0份, 虛擬: 4份)
├── 白飯 (實際: 7份, 虛擬: 20份)  
├── 荷包蛋 (實際: 0份, 虛擬: 7份)
└── 蔬菜盤 (實際: 0份, 虛擬: 2份)

成品可製作:
└── 打拋豬飯 (虛擬庫存: 2份 - 現在受限於蔬菜盤)
```

### 案例特點說明：
✅ **彈性製作**: 可同時使用實際庫存和即時製作  
✅ **準確成本**: 分別追蹤實際成本和虛擬成本  
✅ **智能警告**: 當虛擬庫存不足時提醒補充原料  
✅ **最優利用**: 優先使用實際庫存，減少製作時間
