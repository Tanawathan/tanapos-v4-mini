## 🏗️ 系統架構特色

### 📊 三層庫存架構
```
原物料 (Raw Materials)
    ↓ 食譜製作
半成品 (Semi-finished Products)  ←─┐
    ↓ 食譜組合                    │ 混合模式
成品/菜單項目 (Finished Products) ←─┘
```

### 🔄 彈性組合模式
- **預製模式**: 原物料 → 半成品 → 成品
- **混合模式**: 原物料 + 半成品 → 成品  
- **直接模式**: 原物料 → 成品

### 💡 智能虛擬庫存
- **實際庫存**: 已製作完成的物品
- **虛擬庫存**: 從原料計算可製作的數量
- **總可用量**: 實際 + 虛擬 = 最大供應能力

### 💰 分層成本計算
- **原料成本**: 直接材料成本
- **半成品成本**: 加工製作成本
- **混合成本**: 支援多層級成本結構

## 🎯 業務案例示範

### 實例：打拋豬飯製作流程
```
原物料層級:
├── 豬肉片 (5kg 庫存)
├── 九層塔 (0.5kg 庫存) 
├── 米 (20kg 庫存)
└── 雞蛋 (10顆 庫存)

半成品層級:
├── 打拋豬 (實際: 2份, 虛擬: 5份)
├── 白飯 (實際: 10份, 虛擬: 20份)
└── 荷包蛋 (實際: 3份, 虛擬: 10份)

成品:
└── 打拋豬飯 (可製作: 5份 - 受限於最少材料)
```

### 智能製作決策
- **優先使用實際庫存**: 減少製作時間
- **自動計算短缺**: 提前預警缺料情況
- **成本最優化**: 混合模式選擇最佳成本方案
