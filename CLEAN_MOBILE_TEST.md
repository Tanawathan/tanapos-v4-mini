# 🧹 乾淨手機版 POS 測試指南

## 📱 CleanMobilePOS 特點

### ✨ 關鍵改進
- **完全基於 SimplePOS**: 使用相同的 `store-complete` 和訂單格式
- **純內聯 CSS**: 最乾淨的實作方式，無外部依賴
- **觸控友好**: 44px+ 最小觸控目標，優化手機體驗
- **簡潔介面**: 減少視覺雜訊，專注核心功能
- **無障礙設計**: 支援鍵盤導航和螢幕閱讀器

### 🛠 技術實作
- **Store**: 使用 `store-complete`（與 SimplePOS 相同）
- **樣式**: 100% 內聯 CSS，無外部樣式表
- **相容性**: 支援所有現代瀏覽器
- **響應式**: 手機優先設計

### 🔧 測試步驟

1. **啟動開發伺服器**:
   ```powershell
   .\start-clean-mobile.ps1
   ```

2. **開啟測試頁面**:
   - 網址: `http://localhost:5173/mobile-clean`
   - 或直接訪問: `http://localhost:5173` 然後導航到 /mobile-clean

3. **測試功能**:
   - ✅ 產品瀏覽和分類篩選
   - ✅ 加入購物車和數量調整
   - ✅ 桌號選擇
   - ✅ 備註輸入
   - ✅ 訂單送出（使用 SimplePOS 格式）

### 🎯 預期結果
- 所有功能正常運作
- 訂單格式與 SimplePOS 相同
- 無空白訂單問題
- 流暢的手機操作體驗

### 🐛 若有問題
1. 檢查瀏覽器控制台錯誤
2. 確認資料庫連線
3. 驗證 Supabase 設定
