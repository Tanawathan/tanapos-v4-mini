# 🍰 KDS 甜點分離功能說明

## 新增功能概述

KDS (廚房顯示系統) 現在支援將甜點類餐點從主餐中分離顯示，確保甜點在主餐完成後才開始製作。

## 🎯 功能特點

### 1. 自動甜點識別
系統會自動識別以下甜點關鍵字：
- 甜點、蛋糕、布丁、提拉米蘇、鬆餅
- 冰淇淋、仙草、甜品、巧克力、芒果
- 椰奶仙草、芒果布丁等

### 2. 分離顯示
- **🍽️ 主餐區域**: 顯示所有非甜點餐品
- **🍰 甜點區域**: 顯示甜點類餐品，標註"主餐後上"

### 3. 製作流程控制
- 甜點項目在主餐未完成時顯示為灰色不可點擊狀態
- 顯示"等待主餐"標籤提醒廚師
- 主餐完成後甜點項目自動變為可操作狀態

### 4. 進度追蹤
- 分別顯示主餐和甜點的完成進度
- 總進度統計所有餐點項目
- 視覺化指示器顯示各區域完成狀態

## 🔧 使用方法

### 廚師操作流程
1. **開始製作**: 點擊"開始製作"按鈕
2. **完成主餐**: 逐一點擊主餐項目標記為完成（會顯示刪除線和勾號）
3. **製作甜點**: 主餐全部完成後，甜點區域變為可操作
4. **完成甜點**: 點擊甜點項目標記為完成
5. **整單完成**: 所有項目完成後點擊"全部完成"

### 視覺指示說明
- ✅ **綠色勾號**: 已完成項目
- 🔄 **藍色邊框**: 可操作的主餐項目
- 🍰 **粉色邊框**: 可操作的甜點項目
- ⏳ **灰色顯示**: 等待主餐完成的甜點項目
- 🚫 **刪除線**: 已完成並劃掉的項目

## 🔍 測試建議

### 測試訂單範例
建議使用包含主餐和甜點的混合訂單進行測試：

```
訂單 #12345
- 打拋豬飯 (主餐)
- 涼拌青木瓜 (主餐)  
- 泰式檸檬茶 (飲品)
- 椰奶仙草 (甜點)
- 芒果布丁 (甜點)
```

### 測試流程
1. 訪問 KDS 頁面: `http://localhost:5173/kds`
2. 查看訂單是否正確分離主餐和甜點
3. 測試主餐項目的點擊完成功能
4. 確認甜點在主餐完成前無法操作
5. 驗證甜點在主餐完成後可以操作
6. 檢查整體進度統計是否正確
