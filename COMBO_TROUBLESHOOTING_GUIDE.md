# 🔧 套餐系統問題診斷和修復指南

## 問題症狀
您在點餐系統中看不到套餐選項，儘管管理介面顯示套餐已經存在。

## 解決步驟

### 步驟 1: 執行完整修復腳本
1. 登入 **Supabase Dashboard**
2. 進入 **SQL Editor**
3. 複製並執行 `fix-combo-system-complete.sql` 檔案中的所有內容
4. 確認執行結果顯示 "✅ 套餐系統檢查和修復完成！"

### 步驟 2: 驗證資料庫設置
執行後您應該看到類似以下的結果：
```
檢查項目        | 數量
套餐總數        | 3
選擇規則總數    | 11
```

### 步驟 3: 重新整理應用程式
1. 回到您的 POS 系統
2. **完全重新整理頁面** (Ctrl+F5 或 Cmd+Shift+R)
3. 點選 "🍽️ 套餐組合" 按鈕
4. 確認可以看到套餐列表

### 步驟 4: 測試套餐功能
1. 嘗試點選 "夏日特色套餐"
2. 確認套餐選擇器介面正常開啟
3. 測試選擇不同的前菜、主餐、飲品、甜點
4. 確認可以成功加入購物車

## 可能的問題原因

### 原因 1: 資料表權限問題
- **症狀**: 控制台顯示權限錯誤
- **解決**: 上述腳本會重新設置 RLS 政策

### 原因 2: 資料表不存在
- **症狀**: 控制台顯示 "table doesn't exist"
- **解決**: 腳本會檢查並創建必要的資料表

### 原因 3: 資料匯入失敗
- **症狀**: 資料表存在但沒有資料
- **解決**: 腳本會清除並重新匯入資料

### 原因 4: 瀏覽器快取問題
- **症狀**: 程式碼更新但介面沒有變化
- **解決**: 強制重新整理頁面 (Ctrl+F5)

## 檢查清單

- [ ] 執行 `fix-combo-system-complete.sql` 腳本
- [ ] 確認資料庫中有 3 個套餐和 11 個選擇規則
- [ ] 強制重新整理 POS 頁面
- [ ] 確認套餐按鈕顯示數量 (應該是 "🍽️ 套餐組合 (3)")
- [ ] 測試點選套餐並確認選擇器開啟
- [ ] 測試完整的套餐訂購流程

## 仍然有問題？

如果執行上述步驟後仍然看不到套餐：

1. **檢查瀏覽器控制台**:
   - 按 F12 開啟開發者工具
   - 查看 Console 標籤是否有錯誤訊息
   - 查看 Network 標籤確認 API 請求是否成功

2. **檢查資料庫連線**:
   - 確認 Supabase URL 和 API Key 正確
   - 確認網路連線正常

3. **重新啟動開發伺服器**:
   ```bash
   # 停止當前伺服器 (Ctrl+C)
   npm run dev
   ```
